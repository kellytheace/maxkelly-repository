$$1+2+\cdots+n=\frac{n(n+1)}{2}\,.$$

For multiples of a positive integer $d$ strictly below a limit $L$, the multiples are $d, 2d, \dots, kd$ where $$k=\left\lfloor\frac{L-1}{d}\right\rfloor.$$ Their sum is

$$S_d(L)=d\cdot\frac{k(k+1)}{2}\,.$$

Inclusionâ€“Exclusion (3 or 5)

Numbers divisible by $3$ or $5$ are counted by

$$S(L)=S_3(L)+S_5(L)-S_{15}(L),$$

because multiples of $15$ are included in both $S_3$ and $S_5$. For this problem use $L=1000$ (we exclude the limit itself).

Why this is better than a loop

A brute-force loop checks every $i<L$ (time $O(L)$). The closed-form above uses a constant number of arithmetic operations (time $O(1)$), so it scales to very large $L$.

Python Solution


def sumn(limit, d):
    k = (limit - 1) // d
    return d * k * (k + 1) // 2

limit = 1000
result = sumn(limit, 3) + sumn(limit, 5) - sumn(limit, 15)
print(result)
